# 필수 활동 보장과 마지막 단계 수동 카드 설계

## 목표

- 기상·취침 시간 입력은 10분 단위로 제한한다.
- 새 계획의 기본 활동은 아무것도 선택하지 않은 상태로 시작한다.
- `꼭 넣어줘!` 활동은 요청한 주간 횟수를 모두 포함한 시간표만 완성한다.
- 마지막 단계에서 부모가 요일별 `+` 버튼으로 일정을 직접 추가한다.

## 필수 활동 규칙

필수 활동은 일반 활동보다 먼저 전체 조합을 탐색해 배치한다. 평일 2개·주말 3개의 권장 활동 개수는 필수 활동에 적용하지 않지만, 기상·취침 시간과 고정 일정의 충돌 금지는 유지한다. 깨어 있는 전체 시간의 10분 간격 후보를 사용한다. 그래도 요청 횟수를 모두 넣을 수 없으면 누락된 결과를 만들지 않고 생성을 중단하며 조정 방법을 안내한다.

## 수동 카드 추가

각 요일 제목 옆에 원형 `+` 버튼을 둔다. 누르면 해당 요일이 미리 선택된 대화상자가 열리고 제목, 시작 시각, 종료 시각을 입력한다. 시각은 10분 단위다. 기상·취침 범위를 벗어나거나 기존 카드와 겹치면 저장하지 않고 대화상자 안에서 이유를 표시한다. 성공하면 이동·잠금·삭제가 가능한 일반 카드로 즉시 추가하며 브라우저 저장, 방학 전체 보기, PNG/PDF/PPTX에도 반영한다.

## 검증

- 새 기본 활동의 선택·필수 값이 모두 거짓인지 확인한다.
- 필수 활동이 일일 권장 개수를 넘어도 요청 횟수만큼 배치되는지 검사한다.
- 물리적으로 불가능한 필수 활동은 생성 오류가 되는지 검사한다.
- 수동 일정의 정상 추가, 겹침, 생활 시간 이탈을 단위 테스트한다.
- 데스크톱과 모바일에서 `+` 버튼, 대화상자, 추가된 카드를 직접 확인한다.
