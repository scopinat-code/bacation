# 방학한칸 MVP 구현 설계

## 제품 범위

부모가 방학 기간과 생활시간, 반복 고정 일정, 날짜별 특별 일정을 입력한 뒤 아이가 활동 카드와 빈도·선호 시간·꼭 넣기를 고른다. 브라우저 안의 규칙 기반 엔진이 충돌 없는 반복 주간표를 만들고, 가족은 결과를 잠그거나 삭제하거나 다른 시간으로 옮긴 뒤 PNG 또는 인쇄물로 저장한다. 회원가입, 서버, 외부 API, 완료 감시와 보상 기능은 제외한다.

## 구조

- Next.js App Router와 TypeScript로 하나의 모바일 우선 웹앱을 구성한다.
- UI 상태와 도메인 타입을 분리하고 `lib/scheduler.ts`에 순수 함수 일정 엔진을 둔다.
- 일정 엔진은 고정 일정과 이동 완충시간을 먼저 점유한 뒤, 30분 단위 후보 중 선호 시간대·요일 분산·카테고리 연속성·하루 밀도를 점수화해 활동을 배치한다.
- 고정 일정 겹침은 생성 전 오류로 막고, 수용하지 못한 요청은 이유와 조정 제안을 경고로 반환한다.
- 입력과 결과는 localStorage에 자동 저장한다. 개인정보는 별명과 학년 외에 요구하지 않는다.

## 화면 흐름

랜딩 → 기본정보 → 부모 일정 → 아이 활동 → 함께 조정의 5개 상태로 진행한다. 모바일에서는 한 열, 넓은 화면에서는 보조 설명과 입력 카드를 나란히 둔다. 결과는 월~일 카드로 보여주며 고정 일정은 정확한 시각, 유연 활동은 오전·오후 배지로 구분한다. 드래그에 의존하지 않고 잠금·삭제·다른 시간 버튼을 제공한다.

## 오류·출력·검증

날짜 역전, 기상·취침 역전, 고정 일정 중복, 일정 과밀을 각각 명시적인 한국어 안내로 처리한다. PNG 생성이 실패하면 인쇄 기능을 대안으로 안내한다. 인쇄 CSS는 A4 가로를 사용하고 입력·탐색 컨트롤을 숨긴다. 일정 엔진은 정상 생성, 고정 일정 겹침, 생활시간 준수, 선호 시간 반영, 과밀 미배치 시나리오를 자동 테스트한다.
