# 캘린더형 드래그와 기간 특별 일정 설계

## 목표

4단계의 블록 이동 상태와 착지 결과를 Google Calendar처럼 즉시 이해할 수 있게 만든다. 특별 일정은 하루씩 반복 입력하지 않고 시작일과 종료일을 한 번에 등록한다.

## 드래그 경험

- 잠기지 않은 활동 블록 전체가 드래그 표면이다.
- 드래그 중 원래 블록은 자리를 유지한 채 흐려지고, 포인터에는 큰 미리보기 카드가 따라간다.
- 각 요일을 오전과 오후의 두 착지 구역으로 나눈다.
- 착지 구역에는 실제 배치될 시작·종료 시각을 미리 표시한다.
- 빈 시간이 없는 구역은 비활성 상태와 안내 문구를 보여준다.
- 놓기가 끝나면 블록의 요일, 오전·오후, 시작·종료 시각을 함께 갱신한다.
- 고정 일정과 잠긴 활동은 드래그할 수 없고, 기존 잠금·삭제·자동 이동 기능은 유지한다.

## 기간 특별 일정

- 데이터는 `startDate`와 `endDate`를 사용하고 양 끝 날짜를 모두 포함한다.
- 기존 브라우저 저장 데이터의 단일 `date`는 같은 시작일·종료일로 자동 변환한다.
- 입력 화면에는 시작일, 종료일, 1일·2일·3일 빠른 선택을 제공한다.
- 전체 방학 달력, PDF, PPTX는 기간 안의 모든 날짜를 특별 일정으로 대체한다.
- 목록에는 단일 날짜 또는 날짜 범위와 총 일수를 표시한다.

## 오류 처리와 검증

- 종료일이 시작일보다 빠르거나 방학 범위를 벗어난 일정은 등록하지 않는다.
- 드롭할 빈 시간이 없으면 기존 시간표를 변경하지 않는다.
- 일정 엔진 단위 테스트에서 오전·오후 지정 배치와 기간 포함 범위를 검증한다.
- 모바일과 데스크톱 브라우저에서 마우스·터치 드래그, 기간 입력, 전체 달력 반영을 확인한다.
