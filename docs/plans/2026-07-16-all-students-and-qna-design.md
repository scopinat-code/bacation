# 방학한칸 전체 학생 확장 및 Q&A 설계

## 목표

방학한칸을 초등학교 3·4학년 중심 서비스에서 모든 초·중·고등학생이 사용할 수 있는 방학 시간표 서비스로 확장한다. 학교급에 따라 계획의 주도권, 질문, 활동 카드와 시간 정밀도를 바꾸되, 공부·운동·취미·휴식을 함께 담는 균형형 시간표와 PDF·PPTX 내보내기라는 핵심 경험은 유지한다.

## 학교급별 경험

- 초등학생: 부모가 생활시간과 고정 일정을 입력하고 아이가 활동 카드를 선택한다. 오전·오후 중심의 느슨한 배치를 제공한다.
- 중학생: 학생이 주도하며 공부·학원·취미·휴식을 선택한다. 활동은 1시간 단위로 배치한다.
- 고등학생: 학생이 구체적인 과목·활동을 선택한다. 활동은 30분 단위로 배치한다.
- 중·고등학생의 부모 입력 단계는 선택 사항이다. 학생이 기본 계획을 만든 뒤 필요할 때 가족 행사와 필수 조건만 입력받는다.
- 실행 체크, 타이머, 달성률과 같은 학습관리 기능은 포함하지 않는다.

## 공통 결과

모든 학교급에서 Google Calendar 방식으로 블록을 이동하고 직접 일정을 추가할 수 있다. 반복 주간 일정과 방학 전체 일정을 제공하며 PDF와 편집 가능한 PPTX로 내보낸다. 학교급에 맞는 문구와 학년 표기를 결과물에도 반영한다.

## Q&A 게시판

- 로그인 없이 누구나 문의, 불편 신고, 개선 제안을 작성한다.
- 공개 글은 작성 즉시 목록과 상세 화면에 노출한다.
- 비공개 글은 목록에서 내용을 숨기고 작성 비밀번호로만 열람한다.
- 글 비밀번호는 단방향 해시로 저장하고 같은 비밀번호로 수정·삭제한다. 비밀번호 복구는 제공하지 않는다.
- 관리자는 기존 `/admin` 세션으로 모든 글을 보고 답변·수정·숨김·삭제할 수 있다.
- 답변 등록 시 상태가 `답변 완료`로 바뀌며 운영자 답변임을 명확히 표시한다.
- 숨겨진 봇 입력란, 최소 작성 시간, 요청 식별값의 단기 해시 보관과 반복 제한으로 스팸을 줄인다.
- 게시글은 일반 텍스트로만 처리하고 아이 실명, 학교명, 연락처와 개인 일정을 작성하지 말라는 안내를 표시한다.

## 데이터와 보안

Neon PostgreSQL에 Q&A 글, 답변, 공개 여부, 숨김 상태와 스팸 제한 기록을 저장한다. 원문 비밀번호와 원본 IP는 저장하지 않는다. 관리자 인증은 기존 `ADMIN_PASSWORD`와 `ADMIN_SESSION_SECRET`으로 유지한다. 사용자용 일정 데이터는 계속 브라우저에만 저장한다.

## 검증 기준

- 기존 초등학생 저장 데이터가 마이그레이션되어 계속 열린다.
- 각 학교급의 활동 카드와 생성 시간 단위가 다르게 적용된다.
- 공개·비공개 글 작성, 비밀번호 열람·수정·삭제, 관리자 답변·숨김이 동작한다.
- 아이 개인정보가 Q&A 스키마나 통계 이벤트에 자동 수집되지 않는다.
- 타입 검사, 단위 테스트, 프로덕션 빌드와 실제 Neon DB 런타임 흐름을 통과한다.
